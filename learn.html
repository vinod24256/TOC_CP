Model 01

Turing Machine: Formal Definition

A Turing machine is historically the most powerful computational model, forming the basis of modern computer architecture. It is formally defined as a 7-tuple.

  • Q Finite set of states
  • Σ Input alphabet (excluding blank)
  • Γ Tape alphabet (Σ ∪ {□})
  • δ Q × Γ → Q × Γ × {L, R}
  • q₀ Initial starting state
  • q_acc Halting acceptance state
M = (Q, Σ, Γ, δ, q₀, q_acc, q_rej)

// Caesar Implementation
Q = {q₀, q_lo, q_up, q_sk, q_acc}
Σ = Printable ASCII
Γ = Σ ∪ {□}

δ(q_lo, 'a') → (q₀, 'd', R)
Thesis 02

Church-Turing & Computation

The Church-Turing thesis states that a function is effectively calculable if and only if it is computable by a Turing machine.

Computation is tracked using Instantaneous Descriptions (IDs), denoted as α [q] β, capturing the entire tape state and head position at any given step.

Trace: "cat" (k=3)
[q₀] cat
f [q₀] at
f d [q₀] t
f d w [q₀] □
f d w □ [q_acc]

⊢ M accepts "cat"
Limitation 03

The Halting Problem

Turing proved that a general algorithm to determine whether another program will eventually halt does not exist. This is the cornerstone of undecidability.

If we assumed a HaltDetector existed, we could create a Paradox function that loops if the detector says it halts. This leads to a logical contradiction.

def PARADOX(M):
  if HALT(M, M) == "halts":
    loop forever
  else:
    halt

Run PARADOX(PARADOX) → 💥 Failure
