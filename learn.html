Turing Machine: Formal Definition

A Turing machine is historically the most powerful computational model, forming the basis of the modern computer architecture. It is formally defined as a 7-tuple: M = (Q, Σ, Γ, δ, q₀, q_{accept}, q_{reject}).

The 7 Components

  • Q forms the finite set of continuous states.
  • Σ is the input alphabet (all valid input characters, excluding blank).
  • Γ is the tape alphabet (includes input symbols + blank symbol □).
  • δ is the transition function deciding logic: Q × Γ → Q × Γ × {L, R}.
  • q₀ is the starting state where execution begins.
  • q_{accept} and q_{reject} are halting states.
M = (Q, Σ, Γ, δ, q₀, q_acc, q_rej)

Church-Turing Thesis & IDs

The Church-Turing thesis states that a function on the natural numbers can be calculated by an effective method if and only if it is computable by a Turing machine.

We log a computation sequence using Instantaneous Descriptions (IDs), denoted as α [q] β. Here, α is the tape content to the left of the head, q is the current state, and β is the tape content starting from the head to the right.

Recursive Languages (Type 0)

Our Caesar cipher TM is a Total TM (Decider) — it is guaranteed to eventually halt on all inputs. Hence, the language it decides is Recursive.

Computation Trace for "cat"
[q₀] cat
f [q₀] at
f d [q₀] t
f d w [q₀] □
f d w □ [q_acc]

⊢ M accepts "cat"

The Halting Problem Limit

Turing proved in 1936 that a general algorithm to determine whether another program will eventually halt does not exist. This is the Halting Problem, the most famous example of an undecidable language.

Why is it Undecidable?

If we assumed a perfect HaltDetector(Program, Input) existed, we could write a Paradox(Program) that does the exact opposite of what the HaltDetector predicts. If the HaltDetector claims it will halt, the Paradox loops forever. When we run Paradox(Paradox), logic breaks.

def PARADOX(M):
if HALT_DETECTOR(M, M) == "halts":
loop forever
else:
halt
Run PARADOX(PARADOX) → 💥 Contradiction
